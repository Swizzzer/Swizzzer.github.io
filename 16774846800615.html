第三章 类和对象

2023/02/27 posted in  ComputerScience

对象与对象变量

要想使用对象，就必领首先构造对象，并指定其初始状态。然后，对对象应用方法.Java中使用构造器构造新的实例.构造器的名称应与类名相同.例如,Date类构造器的名称为Date.要构造一个Date对象,需要在构造器前面加上new操作符,如下:

new Date()

如果希望构造的对象可以多次使用,那么需要把构造的对象储存在一个对象变量里.

Date birthday=new Date();

一个对象变量并没有实际包含一个对象而仅仅是引用一个对象.因此,在使用对象变量之前必须初始化对象变量.可以为一个对象变量构造一个新对象,也可以引用一个已经存在的对象.例如

    Date birthday=new Date();
    Date deadline;
    deadline=birthday;

或者

    Date birthday=new Date();
    Date deadline;
    deadline=new Date();

类的声明

类的声明的格式如下:

[类修饰符] class 类名
{
    成员变量的声明;// 描述属性
    成员方法的声明;// 描述功能
}

一个简单的类的例子如下:

public class Student{
    private String name;//姓名属性，私有
    private int age;//年龄属性，私有
    private int score;//学分属性，私有
    /**
     *构造函数(构造器)，读取姓名和年龄属性
     */
    public Student(String name,int age){
         this.name = name;
         this.age = age;
    }
    /**
     *定义一个学习方法。每学习一次学分加1
     */
    public void study(){
        score++;
        System.out.println(name+"正在学习中，学分是"+score);
    }
}

构造器不写返回数据类型.构造器总是伴随着new操作符的执行被调用,因此不能对一个已经存在的对象调用构造器来达到重新设置实例域的目的.例如,使用

 Student Kunihiro=new Student("Kunihiro Hane",28);

创建实例后,执行以下语句将产生编译错误

Kunihiro.Student("Minoru",30);

构造器的注意事项

  • 构造器与类同名
  • 每个类可以有一个以上的构造器
  • 构造器可以有0个、1个或多个参数
  • 构造器没有返回值
  • 构造器总是伴随着new操作一起调用

当参数名和属性名相同的时候，Java会采用就近原则优先使用参数；如果要区分开来，那么可以使用this来区分，加上this.的表示属性，否则表示参数。
例子:

public class Student{
    private String name;//姓名属性，私有
    private int age;//年龄属性，私有
    private int score;//学分属性，私有
    /**
     *构造函数，读取姓名和年龄属性
     */
    public Student(String name,int age){
         this.name = name;
         this.age = age;
         System.out.println("有参数的构造方法");
    }
    /**
     *构造方法的重载
     */
    public Student(){
         System.out.println("没有参数的构造方法");
    }
    /**
     *定义一个学习方法。每学习一次学分加1
     */
    public void study(){
        score++;
        System.out.println(name+"正在学习中，学分是"+score);
    }
    public void study(String book){
        System.out.println(name+"正在学习《"+book+"》中的内容");
}
}

子类的构造-继承

Java使用关键字来实现继承.子类不能直接extends继承父类的构造方法,子类使用super语句继承父类的构造方法.

静态成员

在非静态成员方法中可以访问静态成员方法/属性,反正则不能.即,static声明的方法不能访问非static的操作,非static声明的方法可以访问static声明的属性/方法.

在下面两种情况下使用静态方法:

  • 一个方法不需要访问对象状态，其所需参数都是通过显式参数提供(例如:Math.pow).
  • 一个方法只需要访问类的静态域

对数组静态赋初值时,必须在定义数组的同时就为数组元素分配空间并赋初值.
